# 提示词缓存

提示词缓存可以减少重复上下文带来的成本和延迟。它通常由模型 provider 支持，OpenClaw 负责尽量组织稳定上下文。

适合缓存的内容：

- 系统提示词。
- 长期规则。
- 稳定工具 schema。
- 不常变化的项目说明。

不适合缓存的内容：

- 当前用户消息。
- 临时密钥。
- 刚生成的工具结果。

## 新手提醒

缓存不是让模型“记住秘密”。敏感信息不要为了省钱而放进稳定上下文。
真正适合缓存的是长期不变、可公开给模型看的规则和说明。

如果 provider 支持缓存，长系统提示词和工具 schema 通常最值得优化。
