# 转写与历史卫生

转写和历史记录会影响隐私、成本和回答质量。

建议：

- 不把敏感内容无脑写进长期记忆。
- 群聊上下文按 allowlist 控制。
- 语音转写结果要按通道权限处理。
- 导出 trajectory 前先脱敏。
- 旧会话按策略修剪。

继续阅读：[会话修剪](/tutorials/concepts/session-pruning)、[轨迹导出](/tutorials/tools/trajectory)。

## 新手比喻

会话历史像一本工作笔记。笔记太乱，Agent 会找不到重点；笔记里有密码，导出时就会泄漏。

良好习惯：

- 日志先裁剪再发送。
- 密钥不要发到聊天里。
- 群聊和私聊分清权限。
- 导出前先检查是否含个人信息。
