# Hooks

Hooks 让插件在关键生命周期点插入逻辑，比如 prompt 构建前、工具结果处理后、消息发送前。

Hooks 很强，也容易造成难排查问题。

使用原则：

- 做小事。
- 可关闭。
- 记录清楚。
- 不偷偷改变安全边界。

## 新手怎么理解

Hook 像门口的小开关：消息进来、工具返回、准备发送回复时，它能顺手做一件事。

适合做：

- 记录审计日志。
- 给消息补 metadata。
- 对输出做平台适配。

不适合做：

- 长时间阻塞。
- 静默执行危险命令。
- 在用户不知情时转发敏感内容。
