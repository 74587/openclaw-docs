# Inworld

Inworld 偏向角色、互动和对话体验场景。OpenClaw 可在对应插件或 provider 支持下接入相关能力。

适合：

- 角色型 Agent。
- 实时互动。
- 对话体验实验。

首次使用前，先确认你的 Inworld 项目、API Key 和模型/角色配置已经在平台侧可用。

## 新手怎么理解

普通 LLM provider 更像“问答模型”；Inworld 更偏“角色和实时互动体验”。
如果你只是想让 OpenClaw 正常聊天，先用常规模型 provider。等聊天、通道、Gateway 都跑稳了，再接 Inworld 这类体验型能力。

排查时先看平台侧项目是否可用，再看 OpenClaw 日志。
